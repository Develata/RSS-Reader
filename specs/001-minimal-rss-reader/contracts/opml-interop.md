# OPML 互操作契约

## 目标

定义 MVP 阶段支持的 OPML 导入导出边界，确保用户可以与其他 RSS 阅读器交换订阅源。

## 支持范围

- 导入 OPML 2.0 常见结构
- 导出兼容常见阅读器的 OPML 文件
- 支持读取 `xmlUrl` 作为订阅地址
- 支持读取或导出 `title` / `text` 作为展示名称
- 支持读取或导出分组层级

## 明确不支持

- 通过 OPML 交换已读状态、收藏状态或文章正文
- 依赖 OPML 保留项目内部专属偏好设置
- 基于 OPML 做差量同步或冲突合并

## 导入规则

1. 以 `xmlUrl` 作为订阅源唯一识别依据。
2. 已存在的订阅源不得重复创建。
3. 缺少 `xmlUrl` 的节点忽略。
4. 分组信息可映射为本地 `folder`。

## 导出规则

1. 导出全部有效订阅源。
2. 使用可兼容的 OPML 结构表达订阅源和分组。
3. 不导出本地文章、阅读状态、收藏状态和偏好设置。
