# US2 性能检查模板：10,000 篇文章下的状态切换与搜索

## 记录信息

- 验证日期：`2026-03-25`
- 验证环境：`WSL Ubuntu + Dioxus Desktop (WSLg)`
- 构建方式：`真实桌面会话功能验证 + cargo test -p rssr-infra --test test_entry_large_dataset_performance -- --nocapture`
- 文章总量：`10,000（本地 SQLite 压测样本）`
- 未读文章量：`约 6,660（压测样本 + 手工验证样本）`
- 收藏文章量：`约 2,000（压测样本 + 手工验证样本）`

## 建议前置条件

- 本地数据库内已有约 10,000 篇文章。
- 数据中同时包含已读、未读、已收藏和未收藏文章。
- 至少准备 3 个可命中的标题关键字和 1 个无结果关键字。

## 建议记录方式

- 状态切换与筛选切换至少执行 10 次，记录是否出现明显停顿或批量错乱。
- 搜索建议记录首个结果稳定出现所需时间。
- 如不能精确计时，也要记录“无明显停顿 / 轻微停顿 / 明显停顿”。

## 检查步骤

1. 打开文章页，在默认“全部文章”视图下连续滚动，确认大数据量下列表仍可交互。
2. 选择一篇未读文章切换为已读，记录状态反馈时间。
3. 选择一篇未收藏文章切换为收藏，记录状态反馈时间。
4. 连续对 10 篇文章执行已读切换，确认 UI 和持久化结果一致。
5. 连续对 10 篇文章执行收藏切换，确认 UI 和持久化结果一致。
6. 启用“仅未读”筛选，记录列表稳定更新时间。
7. 启用“仅收藏”筛选，记录列表稳定更新时间。
8. 在筛选开启状态下输入高命中关键字，记录搜索结果更新时间。
9. 输入低命中关键字和无结果关键字，确认无异常卡顿或错误。
10. 进入阅读页，使用快捷键 `M`、`F` 连续切换状态后返回列表，确认状态同步正常。
11. 重启应用后抽查多篇文章，确认状态持久化无误。
