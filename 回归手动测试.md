
## Web 回归清单

1. 首页与路由
- 打开 `/`
- 点击 `首页 / 订阅 / 文章 / 设置`
- 确认页面切换正常，没有白屏或卡死
- 浏览器刷新后，当前页面还能正常重新进入

2. 订阅管理
- 在“订阅”页添加一个有效 RSS URL
- 确认订阅列表出现该 feed
- 点一次 `刷新全部`
- 如果某个 feed 行有 `刷新此订阅`，也点一次
- 如果有 `删除订阅`，测试两步确认删除是否正常

3. 文章列表与阅读
- 进入“文章”页
- 确认能看到文章列表
- 点开一篇文章进入阅读页
- 确认标题、来源、时间、正文都正常
- 返回文章页后列表仍正常

4. 状态操作
- 点一次 `标记已读`
- 点一次 `收藏`
- 开关 `仅未读`
- 开关 `仅收藏`
- 输入标题关键词搜索
- 刷新页面后确认这些状态仍能正确显示

5. 主题切换
- 去“设置”页
- 用主题下拉选择一个预置主题，点 `载入所选主题`
- 确认页面样式立刻变化
- 用主题卡片点一次 `使用这套主题`
- 再点当前主题卡片的 `移除这套主题`
- 确认只移除了当前主题，不是异常清空别的内容
- 点 `导出当前 CSS`，确认能导出
- 如方便，再点 `导入主题文件` 导回一份 CSS

6. 设置持久化
- 在“设置”页改一个普通设置，比如主题模式、刷新间隔或字号
- 点 `保存设置`
- 浏览器刷新
- 确认设置仍保留

7. 配置交换
- 到“订阅”页测试：
- `导出配置`
- `导出 OPML`
- 如方便，再做一次 `导入配置` 或 `导入 OPML`
- 确认没有报错，列表状态正常

8. Web 持久化
- 关闭当前标签页
- 重新打开服务地址
- 确认订阅、设置、自定义 CSS 仍在
- 这一步是 Web 端最关键的本地持久化检查

**重点观察**
- 主题按钮点击后是否“立刻变样式”
- 刷新页面后数据是否还在
- 删除订阅和移除主题是否只影响目标对象
- Console 里是否有新的业务错误
