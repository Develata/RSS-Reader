# US3 手工验证清单：配置交换与导入导出

## 适用范围

- JSON 配置包导入导出
- OPML 导入导出
- WebDAV 上传下载
- 配置交换后订阅与设置恢复

## 环境准备

- 本地至少已有 2 个订阅，其中 1 个带 `folder` 分组。
- 本地设置至少修改过主题、列表密度或刷新间隔中的一项。
- 准备一个可写 WebDAV 路径，或准备另一个全新本地环境用于导入验证。

## 验证步骤

1. 导出配置包 JSON，确认文件包含订阅源和设置，但不包含文章正文、已读或收藏状态。
2. 将导出的配置包重新导入当前环境，确认不会创建重复订阅。
3. 在新环境导入配置包，确认订阅源、`folder` 和用户设置恢复。
4. 导出 OPML，确认分组结构和订阅标题可见。
5. 使用导出的 OPML 重新导入，确认缺少 `xmlUrl` 的节点被忽略，已有订阅不会重复创建。
6. 配置 WebDAV 远端后执行上传，确认远端路径生成正确且返回成功。
7. 在新环境执行 WebDAV 下载并导入，确认订阅源和设置恢复。
8. 确认文章缓存、已读状态和收藏状态没有随配置交换被带入新环境。
9. 使用损坏或未知字段的配置包导入，确认应用返回错误而不是静默忽略。

## 通过标准

- JSON 配置包和 OPML 都能完成一次完整导出再导入。
- WebDAV 上传下载能够完成配置交换，且只交换订阅源和用户设置。
- 非法配置输入会被拒绝，不会污染本地数据库。
