# US2 性能检查模板：大数据量筛选与状态切换

## 适用范围

- 文章量较大时的列表滚动、筛选切换、标题搜索
- 已读 / 收藏状态切换和阅读页快捷键回写

- 收藏文章量：

## 前置条件

- 本地数据库内已有较大样本，建议接近 `10,000` 篇文章。
- 数据中同时包含已读、未读、已收藏和未收藏文章。
- 至少准备 3 个可命中的标题关键字和 1 个无结果关键字。

## 记录规则

- 状态切换与筛选切换建议至少执行 10 次。
- 如不能精确计时，也要记录：
  - 无明显停顿
  - 轻微停顿
  - 明显停顿
- 如果结论来自仓储压测或自动化断言，而不是 UI 实测，必须在备注中标明。

## 检查步骤

1. 打开文章页，在默认“全部文章”视图下连续滚动，确认列表仍可交互。
2. 选择一篇未读文章切换为已读，记录反馈时间或体感结论。
3. 选择一篇未收藏文章切换为收藏，记录反馈时间或体感结论。
4. 连续对 10 篇文章执行已读切换，确认 UI 与持久化结果一致。
5. 连续对 10 篇文章执行收藏切换，确认 UI 与持久化结果一致。
6. 启用“仅未读”筛选，记录列表稳定更新时间或体感结论。
7. 启用“仅收藏”筛选，记录列表稳定更新时间或体感结论。
8. 输入高命中关键字，记录搜索结果更新时间或体感结论。
9. 输入低命中和无结果关键字，确认无异常卡顿或错误。
10. 进入阅读页，使用快捷键 `M`、`F` 连续切换状态后返回列表，确认同步正常。
11. 重启应用后抽查多篇文章，确认状态持久化无误。
