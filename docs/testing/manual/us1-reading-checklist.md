# US1 手工验证清单：订阅、列表与阅读

## 环境准备

- 使用已完成迁移的本地数据库启动应用。
- 准备至少一个可访问的 RSS 或 Atom 订阅地址。

## 验证步骤

1. 打开应用，确认默认进入“文章”页，并可切换到“订阅”“设置”页面。
2. 进入订阅页，输入有效 feed URL 并提交。
3. 确认添加成功后输入框被清空，页面显示成功或状态提示。
4. 确认新订阅会立即触发刷新，随后在订阅列表中看到标题或 URL 摘要。
5. 进入文章页，确认可看到刚刚刷新出的文章列表，默认按发布时间倒序展示。
6. 打开任意文章，确认阅读页能展示标题、来源、发布时间和正文。
7. 对仅提供 HTML 正文的文章，确认正文按 HTML 渲染，而不是原始标签文本。
8. 对仅提供纯文本正文的文章，确认正文以可读文本展示。
9. 断网或输入不可用 feed URL，确认订阅页能显示错误提示，且应用未崩溃。

## 通过标准

- 添加订阅、刷新、列表展示和阅读页打开形成连续闭环。
- 错误场景只影响当前操作，不阻断其他页面导航。
- 阅读页对文本正文和 HTML 正文都能正常展示。
