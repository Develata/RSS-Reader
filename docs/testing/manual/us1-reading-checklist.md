# US1 手工验证清单：订阅、列表与阅读

## 适用范围

- 新增或修改订阅添加、刷新、文章列表、阅读页逻辑
- 调整正文解析、HTML 渲染、离线缓存或错误提示

## 环境准备

- 使用已完成迁移的本地数据库启动应用。
- 至少准备 1 个可访问 feed，最好同时包含：
  - 1 个 HTML 正文样本
  - 1 个纯文本正文样本
- 如需覆盖失败分支，再准备 1 个不可访问或非法 feed URL。

## 验证步骤

1. 打开应用，确认默认可进入“文章”页，并能切换到“订阅”“设置”。
2. 在“订阅”页输入有效 feed URL 并提交。
3. 确认输入框清空，页面出现成功或状态提示。
4. 确认新订阅可触发首次刷新，并在订阅列表中出现标题或 URL 摘要。
5. 进入“文章”页，确认文章默认按发布时间倒序展示。
6. 打开一篇文章，确认阅读页展示标题、来源、时间和正文。
7. 对 HTML 正文文章，确认显示的是渲染结果，不是原始标签文本。
8. 对纯文本正文文章，确认以可读文本展示。
9. 断网或输入非法 feed URL，确认当前操作给出清晰错误提示，且应用不崩溃。

## 通过标准

- 添加订阅、刷新、列表展示和阅读页打开形成闭环。
- 错误场景只影响当前操作，不阻断其他页面导航。
- HTML 与纯文本正文都能正常展示。
