# US1 性能检查模板：刷新与阅读

## 适用范围

- 订阅添加、首次刷新、无更新刷新、阅读页进入速度
- 本地缓存阅读和列表滚动的主观响应性检查

## 前置条件

- 本地数据库已完成迁移。
- 至少准备 1 个稳定可访问的 feed。
- 至少有 1 篇可打开的 HTML 正文文章。
- 如要比较冷启动与热路径，执行前先关闭应用并清理明显干扰测量的后台任务。

## 记录规则

- 每个检查项建议执行 3 次。
- 如果没有精确计时工具，至少记录：
  - 无明显停顿
  - 轻微停顿
  - 明显停顿
- 如果结果明显受网络影响，必须在备注中写明。

## 检查步骤

1. 冷启动应用并进入默认文章页，记录从启动到可点击导航的时间或体感结论。
2. 在订阅页添加 1 个有效 feed，记录从提交到列表出现该订阅的时间或体感结论。
3. 对已有订阅执行“刷新全部”，记录从点击到文章列表出现新文章的时间或体感结论。
4. 在无更新情况下再次刷新，确认不会重复写入，并记录是否有明显卡顿。
5. 进入文章页连续滚动 10 秒，记录是否出现掉帧、输入延迟或滚动阻塞。
6. 打开一篇 HTML 正文文章，记录从点击到正文可读的时间或体感结论。
7. 返回文章列表后再打开另一篇文章，记录二次进入阅读页的时间或体感结论。
8. 断网后打开已缓存文章，确认仍可本地阅读，并记录是否有异常等待。

## 通过标准

- 冷启动、订阅添加、刷新和阅读形成连续闭环，过程中无明显长时间停顿。
- 无更新刷新不会制造重复文章，也不会产生明显阻塞。
- 已缓存文章在离线状态下仍可稳定打开。
