# US3 边界验证模板：配置交换与主题生效

## 适用范围

- 配置交换边界
- 主题切换与设置持久化
- 非法输入、重复订阅、越界设置和 WebDAV 路径边界

## 前置条件

- 本地至少有 2 个订阅，其中 1 个带 `folder`。
- 本地设置已修改主题、列表密度、启动视图、刷新间隔或阅读字号中的至少 2 项。
- 准备：
  - 1 份合法配置包
  - 1 份损坏配置包
  - 1 份包含未知字段的配置包
- 准备一个 endpoint 末尾不带 `/` 的 WebDAV 路径，用于覆盖路径拼接边界。

## 检查步骤

1. 将主题分别切换为浅色、深色、跟随系统，确认根样式立即变化且重启后保持。
2. 修改刷新间隔和阅读字号并保存，确认重新打开设置页后值保持一致。
3. 导出 JSON 配置包，确认只包含订阅和设置，不包含文章正文、已读或收藏状态。
4. 导入合法配置包，确认订阅、`folder` 和用户设置恢复正确。
5. 导入包含重复 feed URL 的配置包，确认应用明确报错，不产生重复订阅。
6. 导入包含未知顶层或嵌套字段的配置包，确认应用拒绝导入。
7. 导入越界设置，例如 `refresh_interval_minutes < 1` 或 `reader_font_scale > 1.5`，确认应用拒绝导入。
8. 导出 OPML 并重新导入，确认缺少 `xmlUrl` 的节点被忽略，已有订阅不会重复创建。
9. 使用不带尾斜杠的 WebDAV endpoint 执行上传，确认远端路径仍落在预期 collection 下。
10. 修改本地配置后从 WebDAV 下载，确认覆盖结果符合“最新版本覆盖”模型。
11. 在新环境导入或拉取配置后，确认文章缓存、已读状态和收藏状态不会被带入。

## 通过标准

- 主题切换与偏好设置保存符合 `FR-012` 和 `FR-017`。
- 配置交换严格限制在订阅源与用户设置范围内，符合 `FR-014` 到 `FR-016`。
- 非法输入和边界输入会被拒绝，不污染本地配置。
